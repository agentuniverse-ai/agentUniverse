# 工具(Tool)
同人类使用与创造工具一样，智能体也可以借助工具拓展其能力边界、提升专业性，补充LLM中原本能力的不足。

在复杂的任务场景中，往往需要专业工具提升智能体们的任务表现，这些工具可以是原本任务领域内已有的API或方法，也可以是用户为了解决问题而新创建出来的方法。

agentUniverse提供了自定义工具的标准方法，结合底层的rpc技术组件与标准的http协议，您可以将任何API与本地代码片段注册成标准的工具供agent与其他组件使用。

## 弹性包装器

`ResilientTool` 为任意已注册工具增加可配置超时、幂等感知指数重试、异常类型白名单、结构化错误重试、降级值和线程安全熔断器。复制 `agentuniverse/agent/action/tool/resilient_tool.yaml.example` 到应用目录，并将 `target_tool` 指向已有组件。

非幂等操作默认不会重试，只有显式开启 `allow_retry_non_idempotent` 才允许。连续终态失败达到 `circuit_failure_threshold` 后熔断器打开，在恢复窗口内拒绝调用，之后仅允许一次半开探测。`resilience_state()` 可读取计数器而不改变目标工具返回值。

```yaml
name: resilient_search
metadata:
  type: TOOL
  module: agentuniverse.agent.action.tool.resilient_tool
  class: ResilientTool
input_keys: [input]
target_tool: google_search_tool
idempotent: true
max_attempts: 3
timeout_seconds: 10
circuit_failure_threshold: 5
circuit_recovery_seconds: 30
```

`timeout_seconds` 表示响应截止时间。对于 Python 线程承载的执行，超时只能停止等待，不能强制终止已在目标工具中运行的代码。因此截止时间失败不会自动重试，即使目标已标记为幂等；否则新尝试可能与未结束的工作线程重叠并重复副作用。原生异步目标可将取消传播给可取消 I/O；半开探测被取消时会安全重新打开熔断器，不会永久占用探测槽位。

每个通过熔断检查的包装器调用都会向 `ToolManager` 请求一个全新的目标副本，从而在并发调用时保留框架的“每次调用一个副本”隔离语义；只有包装器的熔断和指标状态会有意共享。
