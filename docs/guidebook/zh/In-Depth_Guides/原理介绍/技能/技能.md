# 技能(Skill)

Skill（技能）是agentUniverse中一种模块化、可复用的能力单元。每个Skill将一组指令（instructions）和允许使用的工具（tools）打包在一起，使智能体能够按需加载专业化能力。

## 核心概念

### Skill的组成
一个Skill包含以下关键要素：
- **指令（Instructions）**：定义在SKILL.md文件中的Markdown格式的详细指令
- **允许的工具（Allowed Tools）**：Skill可以使用的工具列表
- **允许的工具包（Allowed Toolkits）**：Skill可以使用的工具包列表
- **执行模式（Context）**：`inline`（内联）或`fork`（分叉）

### 执行模式

#### Inline模式（默认）
在内联模式下，Skill的指令和工具直接注入到当前智能体的上下文中。智能体获得Skill的工具后可以直接使用。

#### Fork模式
在分叉模式下，系统会创建一个隔离的子智能体来执行Skill的任务。子智能体拥有独立的上下文和工具集，执行完毕后仅返回最终结果，不会影响父智能体的状态。

## SKILL.md文件格式

每个Skill通过一个`SKILL.md`文件定义，包含YAML前置元数据和Markdown正文：

```markdown
---
name: my-skill
description: "这个技能的简要描述"
version: "1.0"
allowed_tools:
  - tool_name_1
  - tool_name_2
allowed_toolkits:
  - toolkit_name_1
context: inline
max_iterations: 10
---

# 技能指令

这里是详细的技能指令内容，使用Markdown格式编写...
```

### 前置元数据字段说明

| 字段 | 必填 | 说明 |
|---|---|---|
| `name` | 是 | 技能名称（与目录名一致） |
| `description` | 是 | 技能描述，供LLM进行语义匹配 |
| `allowed_tools` | 否 | 允许使用的工具列表，支持通配符语法 |
| `allowed_toolkits` | 否 | 允许使用的工具包名称列表 |
| `context` | 否 | 执行模式：`inline`（默认）或`fork` |
| `sub_agent` | 否 | fork模式下使用的自定义智能体 |
| `model` | 否 | 覆盖默认的LLM模型 |
| `max_iterations` | 否 | 最大工具调用轮次（默认10） |
| `version` | 否 | 技能版本号 |
| `user_invocable` | 否 | 是否对用户可见 |
| `disable_model_invocation` | 否 | 防止LLM自动调用 |

## 在智能体中使用Skill

### 配置方式

在智能体的YAML配置中添加`skill`字段：

```yaml
name: 'my_skill_agent'
description: 'An agent with skills'
profile:
  introduction: '你是一个具备多种技能的智能助手'
  target: '帮助用户完成各种任务'
  llm_model:
    name: 'qwen_llm'
    model_name: 'qwen-max'
action:
  skill:
    - 'algorithmic-art'
    - 'mcp-builder'
metadata:
  type: 'AGENT'
  module: 'xxx.agent_instance.my_skill_agent'
  class: 'MySkillAgent'
```

### 包路径配置

在`config.toml`中配置Skill的扫描路径：

```toml
[CORE_PACKAGE]
skill = ['sample_standard_app.intelligence.agentic.skill']
```

## 目录结构

一个典型的Skill目录结构：

```
my-skill/
├── SKILL.md              # 技能定义文件（必需）
├── templates/            # 模板文件（可选）
│   └── template.html
├── reference/            # 参考文档（可选）
│   └── best_practices.md
└── scripts/              # 脚本文件（可选）
    └── helper.py
```

## 内置示例

框架在`sample_standard_app`中提供了两个Skill示例：

### algorithmic-art
生成算法艺术的技能，包含p5.js生成器模板和HTML查看器。

### mcp-builder
MCP服务器开发指南技能，包含Python和Node.js的MCP服务器最佳实践参考文档。

# 总结
Skill系统为agentUniverse提供了模块化的能力扩展机制，使智能体能够按需加载专业化工具和指令，支持内联和隔离两种执行模式。
